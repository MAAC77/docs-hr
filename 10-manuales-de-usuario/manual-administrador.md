# Manual de Administración del Sistema

---

## Propósito

Este manual está dirigido al personal que administra la plataforma.

Su función es ayudarte a:

- revisar el estado general del sistema;
- administrar dispositivos y su integración;
- administrar usuarios y configuraciones;
- consultar reportes globales.

Este documento sirve como portada de navegación. Cada módulo importante tiene su propio manual detallado.

---

## Antes de comenzar

Para usar este manual correctamente, debes:

1. ingresar con una cuenta que tenga el rol `Administrador`;
2. verificar que puedes ver el menú `Sistema`;
3. confirmar que tienes acceso a las pantallas de administración.

Si alguna de estas opciones no aparece, solicita la revisión de permisos antes de continuar.

---

## Ruta general del rol

Cuando ingresas con perfil de administrador, el sistema te dirige al `Dashboard Admin`.

Desde el menú lateral podrás acceder a estas opciones:

1. `Dashboard Admin`
2. `Listado de Dispositivos`
3. `Mapeo de Usuarios`
4. `Registros de Dispositivos`
5. `Usuarios`
6. `Configuraciones`
7. `Reportes Globales`

## Cuándo usar cada manual

- Usa `Dashboard Administrativo` cuando necesites una vista rápida del estado general.
- Usa `Gestión de Dispositivos` cuando debas registrar, editar, probar o sincronizar equipos.
- Usa `Mapeo de Usuarios a Dispositivos` cuando debas relacionar usuarios biométricos con usuarios del sistema.
- Usa `Registros de Dispositivos` cuando necesites revisar registros recibidos desde los equipos.
- Usa `Gestión de Usuarios` cuando debas consultar usuarios, editar datos del sistema o importar usuarios por archivo.
- Usa `Configuraciones del Sistema` cuando debas cambiar parámetros funcionales.
- Usa `Reportes Globales` cuando debas revisar consolidaciones por persona o departamento.

---

## Recomendaciones importantes

- Antes de cambiar configuraciones o sincronizar dispositivos, verifica que estás trabajando sobre el registro correcto.
- Si una acción modifica datos del sistema, revisa el resultado antes de pasar al siguiente paso.
- Si un cambio impacta a muchas personas, realiza primero una revisión visual del registro afectado.
- Si el sistema muestra errores de acceso o permisos, no continúes con pruebas parciales; solicita validación del rol.
